# SonarQube Cloud Integration Setup Guide

## Prerequisites
- GitHub account with repository access
- SonarQube Cloud account (free tier available)

## Setup Steps

### 1. Create SonarQube Cloud Organization & Project
- Visit: https://sonarcloud.io
- Sign in with your GitHub account
- Create a new organization
- Create a new project linked to this repository

### 2. Get Your Credentials
- **Project Key**: Found in SonarQube → Project Settings → Project Key
- **Organization Key**: Found in SonarQube → Organization Settings
- **Token**: Generate in SonarQube → Account → Security → Generate Tokens

### 3. Configure GitHub Secrets
Go to your GitHub repository:
1. Settings → Secrets and variables → Actions
2. Create new secret: `SONAR_TOKEN`
3. Paste your SonarQube Cloud token

### 4. Update Configuration Files

#### sonar-project.properties
Replace placeholders:
```
sonar.projectKey=YOUR_PROJECT_KEY          # From SonarQube
sonar.organization=YOUR_ORGANIZATION_KEY   # From SonarQube
```

### 5. Verify Test Configuration
Ensure your `package.json` has code coverage enabled:
```json
{
  "scripts": {
    "test": "ng test"
  }
}
```

The workflow will automatically run code coverage during tests.

### 6. Push to GitHub
Once configured, the workflow will:
1. Trigger on push to `main` or `develop` branches
2. Run tests with code coverage
3. Upload analysis to SonarQube Cloud
4. Display results in pull requests

## Troubleshooting

### Coverage not detected
- Ensure `coverage/` folder is generated by tests
- Check `sonar-project.properties` paths match your setup

### Token issues
- Verify `SONAR_TOKEN` secret exists in GitHub
- Generate a new token if needed from SonarQube

### Project not found
- Confirm project key matches exactly
- Check organization key spelling
